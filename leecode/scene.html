场景

毒药

现在有1000瓶药水，其中１瓶有毒，一只老鼠喝了在24h后会准时死亡，药水无色无味，如何用最少的老鼠在24h时找出毒药？

参考链接

时间限制为24h，说明只有一次喂老鼠的机会，，先从小举例：1瓶药水-1只老鼠；2瓶药水-1只老鼠；3瓶药水-2只老鼠；4瓶药水-2只老鼠（方案：1只老鼠喝1，1只老鼠喝2，然后2只老鼠一起喝3则简单推断），从中可以看出，老鼠的死与不死能组成几种组合，则最多可辨别几瓶药水。故只需 $2^n \ge 1000$即可，得到n=10

加强题目：若将原题目中的1天该为2天、3天 ...

思路同理：如改为2天，则每只老鼠可能有3种情况:

  1. 第一天死
  2. 第二天死
  3. 两天都不死

那么只需要$3^n > 1000$；；；得到 n = 7

那么现在有10只老鼠，如何去判断有毒药水的那一瓶？使用进制转化

1000瓶药水编号为 0~999，转化为2进制，分别为:

0000000000

0000000001

0000000010

...

将10只老鼠，分别安排在二进制的0号到9号位，是１就喝，是０就不用喝

比如：0000000001 ——0号老鼠喝 0000000101 ——0号老鼠喝、2号老鼠喝

到最后，死亡的老鼠所在的二进制位置１，然后转化为10进制后得到的数字就是有毒药水的编号.

孩子概率

小明有两个孩子，其中一个为男孩，另一个也为男孩的概率

由于没有强调两个孩子的顺序，因此情况有三种，A男B女、A女B男、A男B男，因此另一个也为男孩的概率为1/3

如果强调了两个孩子顺序，即第一个孩子为男孩，则有两种情况，A男B男，A男B女，因此另一个也为男孩的概率为1/2

日期间隔天数

# encoding = utf-8
import datetime

# ---------------------------------------------
d1 = datetime.datetime(2018, 10, 31)
d2 = datetime.datetime(2019, 2, 2)
interval = d2 - d1
print(interval.days)
# ---------------------------------------------
"""
可以先计算出两个日期与1年1月1日之间的天数，再相减
"""


def leap_year(year):
    flag = False
    if year % 4 == 0:
        if year % 100 != 0 or year % 400 == 0:
            flag = True
    return flag


def day_get(year, month, day):
    res = 0
    mon = [31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]
    if leap_year(year): mon[1] = 29
    for i in range(1, year):
        res += 365
        if leap_year(i):
            res += 1
    for i in range(1, month):
        res += mon[i - 1]
    res += day
    return res


def day_count(year1, month1, day1, year2, month2, day2):
    return day_get(year1, month1, day1) - day_get(year2, month2, day2)


if __name__ == '__main__':
    print(day_count(2018, 10, 31, 2019, 2, 2))

自己设计hashmap

python中的dict与java中的hashmap在产生哈希冲突时分别采用的方法是开放寻址法以及链接法；

参考1；参考2

散列表是基于高速存取的角度设计的，是一种典型的时间换空间的做法

负载因子的概念：比如我们存储了70个元素，但是我们可能为这70个元素申请了100个元素的空间，70/100=0.7，这个数字称为负载因子

开放地址优缺点

优点：记录更容易进行序列化、如果记录总数可以预知，可以创建相对来说比较好的hash函数，此时数据的处理效率很高

缺点：存储记录的数目不能超过桶数组长度，超过就要扩容、使用探测序列，有可能计算时间成本过高、当记录的总数规模很大且记录本身size比较大时，空槽占用的空间会导致明显的内存浪费、删除记录时，比较麻烦。

解决冲突的方法：

​ １．线性探查法：冲突后，线性向前试探，找到近期的的一个空位置。缺点是出现堆积现象，并且在查找时可能会很影响查找效率

​ ２．双散列函数法：在位置d冲突后，再次使用另一个散列函数产生一个与散列表桶容量m互质的数c，依次试探(d + n*c)%m，使探查序列跳跃式分布

链接法优缺点

优点：对于记录总数频繁可变的情况，处理比较好（避免的不必要的内存开销）、记录存储在节点中，而节点是动态分配，不会造成内存浪费，尤其适合那种记录本身size很大的情况、删除记录比较方便，直接操作指针就可以了

缺点：由于使用指针，记录不容易序列化操作、如果所有的key-value对可以提前知道，并且之后不会再发生变化时，此时若创建一个比较好的哈希函数，然后使用开放寻址，可能效果比较好

经常使用的构造散列函数的方法

１．直接寻址法：即H(key) = a*key + b

２．数字分析法：比方一组员工的出生年月日，发现年月日的后几位的区别很大，因此可以散列地址

３．平方取中法：取keyword平方后的中间几位作为散列地址

４．折叠法：将keyword切割成位数同样的几部分，最后一部分位数能够不同，然后取这几部分的叠加和（去除进位）作为散列地址

５．随机数法：选择一随机函数，取keyword的随机值作为散列地址，通经常使用于keyword长度不同的场合

６．除留余数法：取keyword被某个不大于散列表表长m的数p除余后所得的余数作为散列地址。即 H(key) = key MOD p, p<=m。不仅能够对keyword直接取模，也可在折叠、平方取中等运算之后取模。对p的选择非常重要，一般取素数或m，若p选的不好，easy产生同义词

菜品推荐，如何做特征、选模型

１．首先明确任务的类型：菜品推荐，TOPN推荐问题

２．召回推荐排序：

​ 当前这一步，我们需要做的就是找到当前用户的候选推荐菜品集；可以首先计算菜品与菜品之间的相似度，这个可以通过点菜日志中菜品与菜品之间的共现矩阵向量计算，同时菜品是否属于同一菜系【菜的成分信息】都可以作为相似度计算的因素

​ 假设当前用户并不是一个新用户，那么我们有用户的点菜日志信息；那么我们就可以得到当前用户１周内或者１个月内的点菜信息，找出与历史点菜信息最相似的菜品

​ 或者在为用户进行TOPN推荐时，并不仅仅计算最相似的菜品，可以对用户构建模型，特征可以有点餐时间、推荐菜品热度、推荐菜品与历史点餐菜品的相似度、菜品价格、用户所在地区、菜品所属菜系地区等；利用LR模型做一个0~1分类

外卖配送整个流程，有什么思路

首先应该本着一个目标，外卖配送就是要快

大方面应该考虑3部分，餐厅的出餐时间预估、骑手距离用户的路径规划及时间预估、骑手交付到用户手中的驻留时间预估【比如有的写字楼在高峰时间段要等很长时间电梯】

关于出餐时间预估：模型【回归树】可以想到的特征有当前餐厅的订单量、订餐时间、是否是节假日或者周末，去拟合历史出餐时间

关于第二部分，可以考虑的因素有骑手的骑行能力，天气情况，地理路况，同时还要考虑一些突发状况，比如说车坏了，出交通事故了等，去拟合历史的在当前这些情况下骑手的骑行速度；模型【回归树】

关于第三部分：在该区域的历史交付时间，该区域的地理位置，是否是上班写字楼、是否是学校、是否是周末，楼层高度、是普通小区还是高档小区

除此之外，还有很多其他的因素要考虑，比如一些火锅的外卖必须要由专门的火锅配送员进行配送

上面这三部分应该是环环相扣的，应该要形成一个环，考虑每一部分的时间，进行配送员调度

老虎与羊

  • 有一个岛，岛上有 n 只老虎和一只羊，老虎会吃羊，但吃完后会变成羊，请问，这只羊最后能活下来吗？

    条件：

  • 老虎生存为第一要义

  • 老虎在生存的前提下尽可能的吃羊

我们逆向来思考这个题，情形一:只有一只羊和一只狼。很简单，狼会吃掉羊。羊死。 情形二:两只狼一只羊。结果是谁都不敢吃，因为谁吃掉这只羊下一个死的就会是他。羊活 。 情形三:三只狼一只羊。 会吃，因为当其中一只狼吃掉它，就又回到了情形二，即两只狼一只羊。已推出情形二下羊活，所以其中一只狼能吃掉羊，且变为羊后可以存活。羊死。 情形四:四只羊狼和一只羊。四只狼没有一只敢吃掉羊，因为吃掉羊后，将变为情形三，已推出情形三下羊死，所以狼在吃掉羊变为羊后必死，而每一只狼都是理性的，所以没有狼会采取行动。羊活。 情形五:…… 如此推下去会发现，当狼的数量为奇数时羊死，偶数时羊活。所以一百只狼的情形下，也就是情形100，羊活。数目不会有变化。

英文纠错

１．Non-word纠错，即单词拼写错误；首先找到与错词相似的一些候选词，然后从中选出正确的纠错词，候选词可以通过编辑距离找到，然后通过噪声信道模型纠错公式计算各个候选词的概率

２．Real-word纠错，即每个词都是正确的，但是组合在一起成为短语或句子时意思却不对；可以通过语言模型计算不同候选句的概率，例如n-gram语言模型；

中文纠错

１．Non-word纠错；中文的错误通常是字打错了，但是发音是没错的，因此可以通过以拼音为基础，编辑距离等其他方式为辅的方式

２．多个词组成的query纠错；首先对query分词，然后查找每一个候选词的集合；然后也是通过语言模型，计算各个候选句的概率

包含query中所有单词的最短连续窗口的长度【文档以倒排索引给出】

​ 文档中查询ABC三个词，找最短的包含ABC的序列，给的输入是倒排索引的形式（A/B/C出现的位置数组）

​ １．假如要求ABC按顺序出现怎么做

​ 首先分别设置指向ABC的指针，初始化指针为a的最小值，b大于a的最小值，c大于b的最小值，【在以后的每一步指针的移动过程中，都记录窗口的最小值】，增大a，直到大于b，然后增大b，直到大于c，然后增大c，再接着最初的增大a...，直到c不能再增大

​ ２．不按顺序出现怎么做

​ 首先分别设置指向ABC的指针，初始化分别为a/b/c的最小值，【在以后的每一步指针的移动过程中，都记录窗口的最小值】，最小值开始向前移动，移动到自己不再是最小值，这时更新最大最小值，然后像开始那样最小值继续移动，直到a/b/c都到了最大值

五局三胜还是三局两胜

​ 假定有甲、乙两个乒乓球运动员参加比赛，已知甲的实力强于乙。现有两个备选的竞赛规则，”三局两胜制”和”五局三胜制”。请问，这种情况下：选手甲应该争取哪一种规则对自己有利？

​ 比较在两种规则下甲获胜的概率:

​ "三局两胜"

​ $f(p)=p^2+2(1-p)p^2=p^2(3-2p)$

​ "五局三胜"

​ $g(p)=p^3+3(1-p)p^3+C_4^2(1-p)^2p^3$

​ 然后作差：

​ $g(p)-f(p)=3p^2(p-1)^2(2p-1)$

​ 由于甲的实力强于乙，即P > 1/2，因此五局三胜对甲有利

    E = 0.5(E+1) + 0.25*2 + 0.25(E+2) ＝》E=6

  • 平均抛多少次硬币，才会出现连续n个正面的情况

    结果为$2^{n+1} -2$，参考

  • 一堆硬币，每天都随便捡一枚抛，如果捡到正面，就把它翻过来；如果捡到反面，就再抛一下，问很长很长时间以后，硬币正面和反面的比例会趋近于多少？

    x*0 + (1-x)*0.5 = x

    所以 x = 1/3，因此硬币正面和反面的比例会趋近于 x/(1-x) = 1/2

  • 连续抛硬币，直到第一次出现连续两次正面为止，恰好抛了N次的概率是多少？

    • 假设 f(x) 表示第一次出现连续正面的时候，已经抛了 x 次，并且整个过程的第一次抛出的结果是反面；
    • 假设 g(x) 表示第一次出现连续正面的时候，已经抛了 x 次，并且整个过程的第一次抛出的结果是正面。

    　　　所以 f(1)=f(2)=0，g(1)=0，g(2)=1，而当 x>2，

    • 求 f(x+1)，因为第一次是反面，所以这新添加的第一次不影响结果，因此 f(x+1)=f(x)+g(x)

    • 求 g(x+1)，因为第一次是正面，必须要保证第二次不能为正，所以 g(x+1)=f(x)

      于是得到：

      f(x+2)=f(x+1)+g(x+1)=f(x+1)+f(x)

      g(x+1)=f(x)

    • 得到 f(N)，也就得到了 g(N)，而总抛的可能性共有 2^N 次方，因此，概率为：

      (f(N)+g(N))/2^N

  • 抛硬币 N 次，出现连续 M 次正面的概率是多少？

    参考，对于一般的扔n次硬币，对于出现k次正面的情况，只要计算$(1 - x) / (1-2x+x^{k+1})$在$x^{n+1}$处的泰勒展开系数$a{n+1}$，然后概率为$1 - a{n+1}/2^n$

  • 抛N次硬币，正反两面出现次数相同的概率是多少？

    $C(n, n/2) / 2^n$；

    正面超过反面的概率？

    由于正面超过反面的概率和反面超过正面的概率相同，因此结果为１－上面那一问的结果之后除以2

  • 抛硬币，正面继续抛，反面不抛。问抛的次数的期望

    设期望为E，整个过程可以分为两部分；第一次正面，此时为0.5 * 1，第一次为反面，此时为0.5 *(E + 1)；即E = 0.5 * 1 + 0.5 *(E + 1)，得到 E = 2

参考链接

  • 只有一个鸡蛋时，只能从下到上一层层的测试

  • 当有两枚鸡蛋时，求扔的测试总次数

    常规的解法是用动态规划，分割成的子问题是最优解的时候，原问题也是最优解；一共有N层楼，对应扔的总次数为f(N)，假设第一次从x层向下扔碎了，接下来就只能一层层的试验，最多从第1层到第x-1层需要试验x-1次，加上扔第一个鸡蛋那一次，总的次数就是(x-1)+x=x次；当一次没碎的情况下，相当于一个新的相似子问题：在N-x层中寻找新的扔鸡蛋的次数f(N-x)，因此总的次数就是f(N-x) + 1

    于是：f(x) = max(x, f(N-x)) 其实感觉这个问题很难理解

    """
大楼扔鸡蛋问题，当有两个鸡蛋时
"""
def dropegg(n):
    # 初始化dp数组
    dp = [float('inf') for _ in range(n+1)]
    # 没有楼层或者只有一层时
    dp[0], dp[1] = 0, 1
    for i in range(2, n+1):
        for x in range(1, i+1):
            # x代表从第几层开始向下扔
            res = 1 + max(x-1, dp[i-x])
            dp[i] = min(dp[i], res)
    print(dp[n])

    """
从两个鸡蛋扩展到m个鸡蛋的问题
"""
def dropEgg(n, m):
    # 初始化dp数组
    # dp[i][j]代表i个鸡蛋，第一次在j层楼测试，然后一共j层需要的次数
    dp = [[float('inf') for _ in range(n+1)] for _ in range(m+1)]
    # 没有楼层，或者只有一层楼时，
    for i in range(m+1):
        dp[i][1] = 1
        dp[i][0] = 0
    # 只有一个鸡蛋
    for j in range(n+1):
        dp[1][j] = j
    for i in range(2, m+1):
        for j in range(2, n+1):
            for x in range(1, j+1):
                res = 1 + max(dp[i-1][x-1], dp[i][j-x])
                dp[i][j] = min(dp[i][j], res)
    print(dp[m][n])
