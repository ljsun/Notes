基本数据类型数组排序

import java.util.Arrays;
int[] a = {1,4,-1,5,0};
// 对于基本数据类型的数组
Arrays.sort(a); // 数组a[]的内容变为{-1,0,1,4,5}

// 如何进行降序排列 - 方法１
Arrays.sort(a, Collections.reverseOrder());

// 如何进行降序排序 - 方法２
// 将int类型数组转化为Integer数组
Integer[] a = {9, 8, 7, 2, 3, 4, 1, 0, 6, 5};
// 定义一个自定义类MyComparator对象
Comparator cmp = new MyComparator();
Arrays.sort(a, cmp);
// 定义MyComparator类，实现Comparator接口
class MyComparator implements Comparator<Integer>{
  @override
  public int compare(Integer o1, Integer o2) {
    /*
    如果o1小于o2，我们就返回正值，如果o1大于o2，我们就返回负值
    这样颠倒一下，就可以实现降序排列了，反之即可自定义升序排列
    即正值就颠倒
    */
    return o2-o1;
  }
}

java-封装/继承/多态/抽象/接口

参考链接２

封装

​ 隐藏对象的属性和实现细节，仅对外公开接口；属性私有化，对每个属性提供getter和setter方法，并且通常在类初始化时通过构造函数对属性赋值

继承

​ 父类的私有属性和构造方法不能被继承，子类可以写自己特有的属性和方法，目的是实现功能的扩展；子类构造时先执行父类构造函数，再执行子类构造函数；在子类中使用super关键字访问父类对象

多态

​ 实现多态的三个必要条件为：继承、重写、向下转型；多态的实现方式：基于继承实现的多态；基于接口实现的多态

抽象

​ 抽象类提供了继承的概念，它的出发点就是为了继承，否则它没有存在的任何意义。所以定义的抽象类一定是用来继承的；抽象类不能被实例化；抽象方法必须由子类进行重写；只要包含一个抽象方法的抽象类，该类必须要定义为抽象类；抽象类可以包含具体的方法，也可以不包含抽象方法；abstract不能与final并列修饰一个类；abstract不能与private static final或naive并列修饰同一个方法；子类不需要重写父类的所有抽象方法

接口

​ 接口更类似是一种编写规范；一个接口可以同时被多个类实现，并且一个类也可以同时实现多个接口，这就在一定程度上弥补了抽象类不能多重继承的缺陷；一个类实现某个接口，就必须实现该接口中的所有方法；接口中的成员变量默认（也只能）是pulic static final，接口中的方法默认是public，也只能是public

​ 抽象类是对类抽象，而接口只是对行为的抽象；抽象类是从子类发现公共部分，然后泛化为抽象类，例如猫、狗可以抽象成一个动物类抽象类、而接口只是定义一个行为，鸟、飞机可以实现飞Fly接口，具备飞的行为，这里我们总不能将鸟、飞机共用一个父类吧！

懒汉式单例【在第一次调用的时候进行实例化】

/**
 * Title：Singleton1<br>
 * Description：单例模式——懒汉式
 * 
 * @author QiuChangjin
 * @date 2018年4月17日
 */
public class Singleton1 {

    private static Singleton1 instance = null;

    private Singleton1() {
    }

    /**
     * 1、适用于单线程环境（不推荐）
     * 如果两个线程同时运行到判断instance是否为null的if语句，并且instance的确没有被创建时，那么两个线程都会创建一个实例，此时类型Singleton1就不再满足单例模式的要求了。
     */
    public static Singleton1 getInstanceA() {
        if (null == instance) {
            instance = new Singleton1();
        }
        return instance;
    }

    /**
     * 2、适用于多线程环境，但效率不高（不推荐），很容易造成线程阻塞
     */
    public static synchronized Singleton1 getInstanceB() {
        if (instance == null) {
            instance = new Singleton1();
        }
        return instance;
    }

    /**
     * 3、双重检查加锁（推荐）
     */
    public static Singleton1 getInstanceC() {
        // 先判断实例是否存在，若不存在再对类对象进行加锁处理
        if (instance == null) {
            synchronized (Singleton1.class) {
                if (instance == null) {
                    instance = new Singleton1();
                }
            }
        }
        return instance;
    }
}

 * Description：单例模式——静态内部类方式
 * 
 * @author QiuChangjin
 * @date 2018年4月17日
 */
public class StaticSingleton {
    /**
     * 私有构造方法，禁止在其他类中创建实例
     */
    private StaticSingleton() {
    }

    /**
     * 获取实例
     */
    public static StaticSingleton getInstance() {
        return StaticSingletonHolder.instance;
    }

    /**
     * 一个私有的静态内部类，用于初始化一个静态final实例
     */
    private static class StaticSingletonHolder {
        private static final StaticSingleton instance = new StaticSingleton();
    }

    /**
     * 方法A
     */
    public void methodA() {
    }

    /**
     * 方法B
     */
    public void methodB() {
    }

    public static void main(String[] args) {
        StaticSingleton.getInstance().methodA();
        StaticSingleton.getInstance().methodB();
    }
}
