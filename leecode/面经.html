  • 如何在n个数组中找出它的中位数（n个数组无法完全放在内存中）

    １、使用外部归并排序，对多个数组先进行排序，全部排序完成后再计算中位数

    ２．利用桶排序的思想先对数据进行排序，然后再计算中位数，具体的做法为：

    ​ (1): 将整型的每1byte作为一个关键字，将10G数据每2G读入一次内存，然后遍历每个数据，取出最高8位，这8bits最多表示256个桶，这样的话可以根据8bit的值来确定丢入第几个桶。最后把每个桶写入一个磁盘文件，同时在内存中统计每个桶内数据的数量

    ​ (2): 根据256个桶内的数量，计算中位数在第几个桶中。很显然，2,684,354,560个数中位数是第1,342,177,280个。假设前127个桶的数据量相加，发现少于1,342,177,280，把第128个桶数据量加上，大于1,342,177,280。说明，中位数必在磁盘的第128个桶中。而且在这个桶的第1,342,177,280-N(0-127)个数位上。N(0-127)表示前127个桶的数据量之和。然后把第128个文件中的整数读入内存。

    ​ (3): 继续以内存中的整数的次高8bit进行桶排序。过程和第一步相同

    ​ (4): 一直下去，直到最低字节(7-0bit)的桶排序结束，相信在这个时候完全在内存中使用一次快排就可以了

    ​ 【上面的桶排序假设的是32位无符号整数，对于有符号的整数，可以先根据最高位将数据分为正数与负数，然后判断中位数在正数区间还是在负数区间，然后再利用桶排序】

    ３．分而治之：基于二进制位映射分割

    ​ 首先根据最高位将文件分为file_0和file_1，判断中位数在哪个文件中（假设在file0中），然后将file0中的文件再按照次高位分为file_0_0与file_0_1，继续判断中位数在哪个文件中 ... 直到内存能够容纳剩余的数

  • 一个盒子里面有m个红球，n个白球，从这个盒子里面拿出来x个球，问红球个数大于白球的概率？

    """
一个盒子里面有m个红球，n个白球，从这个盒子里面拿出x个球，问红球个数大于白球的概率？
"""
def pro(m, n, x, y):
    # y代表红球至少出现的次数
    if y > x:
        return 0
    if y == 0:
        return 1
    if y > m:
        return 0
    if x - n > y:
        return 1
    p1 = pro(m-1, n, x-1, y-1)
    p2 = pro(m, n-1, x-1, y)
    # m/(m+n)代表第一个球是红球的概率、n/(m+n)代表第一个球是白球的概率
    return m/(m+n) * p1 + n/(m+n) * p2
if __name__ == '__main__':
    print(pro(5, 5, 3, 2))
